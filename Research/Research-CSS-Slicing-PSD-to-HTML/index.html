History

  • Personal Computer
    • ( Fixed Size of Screen Monitor )
    • Layout by Float Attributes Since 2000
      • Float Attributes Still Alive in Modern Web Design。
      CORE : Float Automatic Typesetting，It Almost Meet All Layout Needs。
      KEYWORD : float: left | right、clear: both | left | right、Clear Fix。
    • Layout by Table、 Frame Tag Since 2002
      • Newsletter、EDM Still Used Table Layout。(Fast & Intuitive)
      • Frame Tag Used on Sidebar Usually in The Past。
      • Important !!! Frame Tag Not Supported in HTML5。(Substitute Iframe Tag)
  • Modern Web
    • ( Mobile First、Unobtrusive JavaScript、Progressive Enhancement )
    • Layout by Display : Inline-Block Attributes Since 2008
      • CORE : Simpler Than Float Attributes to Align up and down and left and right。
      KEYWORD : display: inline-block、text-align: left | center | right、vertical-align: top | middle | bottom。(Collocation width & height & box-sizing...)
    • Layout by Display : Flex Attributes Since 2012
      • CORE : One-Dimensional Typesetting Tool，The Most Mature and Widely Used Technique in Recent Years，Improve the Defects of Inline-Block in Settings and Provide a Dedicated and Complete Solution。
      KEYWORD : flex: none | auto、flex: none | auto、justify-content: flex-start | center | flex-end、align-items: flex-start | center | flex-end。
    • Layout by Display : Grid Attributes Since 2017
      • CORE : Two-Dimensional，Concept-Like Table-Cutting Ability，and Maintain Complete Design Flexibility，At First, There Were Not Many Browsers That Supported Grid. However, Almost All New-Generation Browsers Have supported It Now. Even IE 11 Supports。
      KEYWORD : display: grid、grid-template: Grid cutting settings、justify-items, align-items、grid-column, grid-row。
  • SPA Since 2003
    • ( Single-Page Application )( Modules、Components )
    CORE : Is a Web Application or Website That Interacts With the Web Browser by Dynamically Rewriting the Current Web Page With New Data From The Web Server, Instead of The Default Method of The Browser Loading Entire New Pages. The Goal is Faster Transitions That Make the Website Feel More Like a Native App。
    KEYWORD : CSS Preprocessor SASS | LESS | Stylus | PostCSS ...、JavaScript Framework React | Angular | Vue ...(Use Preprocessor to JavaScript Code Like Node.js-Webpack)
  • RWD Since 2010
    • ( Responsive Web Design )
    CORE : Is an Approach to Web Design That Makes Web Pages Render Well on a Variety of Devices and Window or Screen Sizes.Appropriate Rendering For Different Resolutions When Browsing。
    KEYWORD : Viewport、breakpoint、Media queries @media、Use Liquidity / Proportional Design System、Use Relative Proportions of Responsive Size Images / Viedos / Text
    SAMPLE :
    // 指定螢幕寬度為裝置寬度，畫面載入初始縮放比例 100%
    <meta name="viewport" content="width=device-width, initial-scale=1">
    // 以下兩種設定都可以防止使用者做畫面縮放，將畫面鎖在縮放比例 100%
    <meta name="viewport" content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=1">
    <meta name="viewport" content="width=device-width, initial-scale=1, user-scalable=no">
    @media (query) { /* CSS Rules used when query matches */ }

    <style> <!-- 當螢幕大小在 400 - 500px 間時使用 --!>
    @media (min-width: 400px) and (max-width: 500px){
    h1 { color: fuchsia; }
    .desc:after { content: " In fact, it's between 500px and 600px wide."; }
    }</style>
    外觀設計模式 : 詳細內容請閱 : Google RWD指南
    主體為流動 (mostly fluid)、欄內容下排 (column drop)、版面配置位移 (layout shifter)、微小調整 (tiny tweaks) 和畫布外空間利用 (off canvas)
  • PWA Since 2015
    • ( Progressive Web Apps )
    參考.1 TechBridge 技術共筆部落格
    詳細內容請閱 : Google PWA指南
    是一種普通網頁或網站架構起來的網絡應用程式，但它可以以傳統應用程式或原生移動應用程式形式展示給用戶。這種應用程式形態視圖將目前最為現代化的瀏覽器提供的功能與行動裝置的體驗優勢相結合。
    KEYWORD : 瀏覽器的「新增到主畫面」的功能讓Web App 成為可能。
    Service Worker以及Manifest、如果你能夠利用上面這兩項技術，讓你的 Web App 成功安裝在手機上並且看起來跟 Native App 沒兩樣，我覺得就能稱作是 PWA。by_huli
  • AMP Since 2016
    • ( Progressive Web Apps )
    詳細內容請閱 : Google AMP指南
    是Google帶領開發的開源專案，目的是為提升行動裝置對網站的存取速度。AMP也可指其衍生的標準和庫等專案成果。AMP在HTML等廣泛使用的網路技術基礎上進行改良， 三大要素AMP HTML、AMP JS、AMP Cache。

05.

absolute 切版

【網頁切版】想併排用float，想重疊用absolute - 尼桑的技術網站
使用時機：想要讓元素進行重疊 圖層關係：CSS與下面圖層愈上面，z-index愈大愈上面 Absolute定位：當下absolute時，元素會去找有設定relative的父元素為座標系定位，而如果都沒有就是以body為定位 缺點：absolute不會撐開父元素，因此如果元素本身太寬，會超出去。

float的缺點就是它不會撐開外部容器，所以需要加上clear讓外部容器正常撐開 如果Ａ元素開始不打算加入併排，就要在Ａ元素的CSS加上clear:both 建議把clear:both單獨寫在class=clearfix的div上，因為假設有n個需要clear，這樣就要必須改n個CSS（超級很麻煩），所以不如直接用div加在HTML上

09.

Flex 兩欄 切版

圖解：CSS Flex 屬性一點也不難| 卡斯伯Blog - 前端，沒有極限

第一次用Flex 切版就上手 - 五倍紅寶石

flex: flex-grow 子元件的伸展性
子元件可以從剩下的空間裡額外得到多少比例的空間。
flex: flex-shrink 子元件的收縮性
父元素的空間不夠放入所有子元件時，整體收縮的比例是多少。
flex-basis 指的則是子物件的最小寬度或高度。
flex-direction 可以決定子元件的排序方向
flex-wrap 則決定了子物件是否可以換行
子物件的對齊設定: justify-content | align-items | align-content

自動使用設定的寬度將 Grid Box 補滿

grid-area 為 Grid Box 內的區塊設置名稱
grid-area 必須連續及不能斷開。只能是 X * Y 的矩形
grid-template-areas 以設定了 gird-area 名稱的區塊填滿每個格子
grid-row & grid-column
將指定區塊放到 Grid Box 的起始線和終點線
